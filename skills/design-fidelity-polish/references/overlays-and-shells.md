# Overlays And Shells

## Parent-State Default

Treat dialogs, drawers, and state-only screens as parent-state interactions unless the user explicitly wants separate routes.

## Viewport Overlay Rule

If the design shows an overlay covering the viewport, implement it at the viewport layer.

## Shell Ownership Rule

Shell-owned surfaces such as notifications or global drawers should usually live in the shell rather than behind dedicated pages when the design implies that ownership.
