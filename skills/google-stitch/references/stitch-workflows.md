# Stitch Workflows

Use the lightest Stitch flow that fits the request.

## Existing Project Or Screens

- inspect the project first
- list the relevant screens
- normalize the screen inventory
- hand off into the shared implementation or polish workflow

## Prompt-Only Starting Point

- create a Stitch project only if the user actually wants Stitch generation
- generate the minimum useful screen set
- review the generated screens before treating them as implementation source
- capture identifiers and caveats for downstream work

## Edit Flow

- edit existing screens only when the source itself needs revision before implementation
- keep the edit intent scoped to the user’s actual goal
