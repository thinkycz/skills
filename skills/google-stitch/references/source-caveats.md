# Source Caveats

Stitch can be a strong visual source, but it is not always enough by itself.

## Ask For More Source Material When Needed

Ask for written specifications, backend contracts, or clarified UX intent when:

- fields or workflows are underspecified
- a generated screen conflicts with existing product rules
- acceptance depends on details not present in the Stitch screens

## Conflict Rule

If Stitch and the written specification disagree in a way that changes implementation, stop and ask the user which one has priority.

## Fidelity Rule

If Stitch output is not precise enough for exact parity acceptance, say so clearly and document the remaining ambiguity.
