# Session Patterns

Use this workflow for large design-to-app builds:

1. source adapter and repo discovery
2. screen inventory and source normalization
3. platform adapter selection
4. phased plan and docs creation
5. first-pass route, page, and state implementation
6. traversability completion
7. fidelity completion
8. verification and release-readiness

Prefer visible forward progress over broad speculative rewrites.

When the source changes midstream, re-baseline the inventory and phase plan instead of patching around stale assumptions.
